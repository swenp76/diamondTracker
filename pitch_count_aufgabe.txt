# Aufgabe: Pitch-Count-System für Jugendspieler implementieren

## Hintergrund
diamondTracker ist eine Android Baseball Coach App (Kotlin, SQLite).
Die App soll das Pitch-Count-System des BSV NRW für Jugendspieler unterstützen.
Der "Pitch Count" basiert auf "Batters Faced" (BF) – nicht auf einzelnen Pitches.

---

## Anforderungen

### 1. Datenbank erweitern (DatabaseHelper.kt)
DB-Version erhöhen. Neue Tabellen/Spalten:

**Tabelle `pitcher_appearances`** (Pitcheinsätze speichern):
```
id INTEGER PRIMARY KEY AUTOINCREMENT
player_id INTEGER          -- Verweis auf players.id
game_id INTEGER            -- Verweis auf games.id  
date TEXT                  -- Datum des Einsatzes (YYYY-MM-DD)
batters_faced INTEGER      -- BF in diesem Spiel
```

**Spalte in `players`** hinzufügen:
```
birth_year INTEGER DEFAULT 0   -- Geburtsjahr des Spielers
```

### 2. Pitch-Count-Regeln implementieren (neue Klasse: PitchCountRules.kt)

Erstelle eine Klasse mit folgenden Regeln:

#### Jahrgang 2013 und jünger:
- Max BF pro Kalendertag: **21**
- Ruhezeiten:
  - 18–21 BF → 4 Kalendertage Pause
  - 14–17 BF → 3 Kalendertage Pause
  - 10–13 BF → 2 Kalendertage Pause
  - 6–10 BF → 1 Kalendertag Pause
  - 1–5 BF → 0 Tage Pause
- Zweites Spiel am selben Tag erlaubt wenn erstes Spiel ≤ 5 BF

#### Jahrgang 2012–2010:
- Max BF pro Kalendertag: **24**
- Ruhezeiten:
  - 21–24 BF → 4 Kalendertage Pause
  - 17–20 BF → 3 Kalendertage Pause
  - 13–16 BF → 2 Kalendertage Pause
  - 9–12 BF → 1 Kalendertag Pause
  - 1–8 BF → 0 Tage Pause
- Zweites Spiel am selben Tag erlaubt wenn erstes Spiel ≤ 8 BF

#### Jahrgang 2009–2007:
- Max BF pro Kalendertag: **27**
- Ruhezeiten:
  - 21–27 BF → 4 Kalendertage Pause
  - 17–20 BF → 3 Kalendertage Pause
  - 13–16 BF → 2 Kalendertage Pause
  - 9–12 BF → 1 Kalendertag Pause
  - 1–8 BF → 0 Tage Pause
- Zwei Spiele am selben Tag immer erlaubt bis max BF erreicht

#### Wichtige Regeln für alle Jahrgänge:
- Ruhezeit beginnt am **Folgetag** des Einsatzes (nicht am selben Tag)
- Ein Spieler darf nur **einmal pro Spiel** als Pitcher eingesetzt werden
- BF werden über mehrere Spiele am selben Tag **addiert**

### 3. Neue Methoden in DatabaseHelper.kt

```kotlin
// BF eines Spielers an einem bestimmten Datum (über alle Spiele)
fun getTotalBFForDate(playerId: Long, date: String): Int

// Alle Pitcheinsätze eines Spielers (für Ruhezeit-Berechnung)  
fun getPitcherAppearances(playerId: Long): List<PitcherAppearance>

// Einsatz speichern
fun savePitcherAppearance(playerId: Long, gameId: Long, date: String, bf: Int): Long

// Ist der Spieler heute einsatzbereit? (Ruhezeit prüfen)
fun isPitcherAvailable(playerId: Long, today: String): Boolean

// Wie viele BF hat der Spieler heute noch übrig?
fun getRemainingBFForToday(playerId: Long, today: String): Int
```

### 4. UI: Pitch-Count-Anzeige in PitchTrackActivity

Zeige während des Spiels an:
- Aktuelle BF des Pitchers heute
- Verbleibende erlaubte BF
- Farbliche Warnung wenn nahe am Limit:
  - Grün: > 5 BF übrig
  - Gelb: 3–5 BF übrig  
  - Rot: 1–2 BF übrig, oder Limit erreicht

---

### 5. UI: Neue Activity "PitchCountActivity"

Erstelle eine neue Activity die für ein Team anzeigt:
- Liste aller Spieler mit Pitcher-Position
- Pro Spieler: Heutiger BF-Stand, max erlaubte BF, Status (verfügbar/Pause)
- Wenn Pause: Wie viele Tage noch
- Einsatzbereit = grüner Indikator, Pause = roter Indikator

Layout: RecyclerView mit CardView pro Spieler.
Im AndroidManifest.xml eintragen.
Von SettingsActivity oder GameDetailActivity erreichbar.

---

### 6. Geburtsjahr bei Spieler-Erstellung abfragen

In TeamDetailActivity / dialog_add_player.xml:
- Neues Feld: "Geburtsjahr" (NumberPicker oder EditText, z.B. 2009)
- Wird in players.birth_year gespeichert
- Wird für die Jahrgangsgruppe und damit die BF-Grenzen verwendet
